# 模板参考

各 Phase 使用的产出模板。

## 需求规格模板 (Phase 1)

```markdown
## 需求规格

### 类型
[新功能 / 功能增强 / Bug 修复 / 重构 / 性能优化]

### 背景
[为什么需要这个需求]

### 功能描述
[清晰描述期望行为]

### 输入
- [输入项1]: [类型和约束]

### 输出
- [输出项1]: [类型和约束]

### 验收标准
- [ ] [标准1]

### 非功能要求
- [性能/安全/兼容性等]

### 约束
- [不能改变的东西]
```

## 领域模型模板 (Phase 2)

```markdown
## 领域模型

### 聚合: [聚合名称]

#### 实体
- [实体名]: [职责描述]
  - 标识: [唯一标识字段]
  - 属性: [关键属性列表]

#### 值对象
- [值对象名]: [描述]

### 领域服务
- [服务名]: [职责]

### 领域异常
- [异常名]: [触发条件]

### 代码结构映射

src/[package]/core/
+-- entities.py       # 实体和值对象
+-- converter.py      # 领域服务接口
+-- exceptions.py     # 领域异常
```
